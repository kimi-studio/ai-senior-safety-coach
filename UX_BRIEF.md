# UX BRIEF — 중장년층 대상 AI 코칭 및 안전 알림 앱 프로토타입

- 작성일: 2026-03-11
- 목적: 위시켓 프로젝트 `153134` 지원용 상세 프로토타입 기획
- 산출물 목표: 인터랙티브 모바일 프로토타입 + 보호자 보조 화면 + 한국어 발표용 기획 페이지

---

## 1. 프로젝트 한 줄 정의

중장년층이 매일 쉽게 사용하는 **AI 생활 코치 앱**에 **복약/체크인/안전 알림**과 **보호자 안심 연동**을 결합한 라이프케어 서비스.

---

## 2. 배경 및 전제

공개 공고 정보 기준으로 프로젝트의 정식 요구사항이 모두 공개되어 있지 않으므로, 아래 브리프는 **프로젝트 제목/예산/기간/타깃 특성 기반의 합리적 가정**을 포함한다.

### 공개 정보 기반 해석
- 타깃: 중장년층
- 핵심 키워드: AI 코칭, 안전 알림
- 플랫폼: 웹/안드로이드/iOS 가능성 검토 필요
- 제안 목적: 사업계획서/신청서에 포함 가능한 구현안과 MVP 범위 제시

### 본 브리프의 전략
- 실제 의료 진단 서비스가 아닌 **생활 중심 AI 코칭**으로 접근
- 안전 기능은 실시간 고도 센서 분석보다 **체크인/알림/예외 대응** 중심 MVP로 설계
- 본 사용자와 보호자 경험을 함께 제시해 설득력 강화

---

## 3. 제품 목표

### 사용자 관점
1. 오늘 무엇을 해야 하는지 쉽게 이해한다.
2. AI 코칭을 부담 없이 받아들인다.
3. 복약/생활 루틴을 꾸준히 유지한다.
4. 불안하거나 위험할 때 빠르게 도움을 요청한다.
5. 보호자와 필요한 만큼만 상태를 공유한다.

### 비즈니스 관점
1. 중장년층도 쉽게 쓰는 UX를 증명한다.
2. 재방문을 만드는 일일 루틴 구조를 만든다.
3. 보호자 연동을 통해 차별점을 확보한다.
4. MVP 이후 구독/기관 제휴로 확장 가능한 구조를 보여준다.

## 4. 핵심 사용자

### Primary Persona A — 본 사용자
- 연령: 55~72세
- 특징:
  - 스마트폰 사용은 가능하지만 복잡한 앱은 부담
  - 건강 관리 필요성을 느낌
  - 카카오톡, 유튜브, 문자, 전화 사용 비중 높음
  - 작은 글씨, 복잡한 메뉴, 긴 가입 절차를 싫어함
- 니즈:
  - 쉽게 이해되는 오늘 할 일
  - 실천 가능한 건강/생활 조언
  - 놓치지 않는 알림
  - 위급 시 빠른 도움 요청

### Secondary Persona B — 보호자
- 연령: 40~60대
- 특징:
  - 부모의 건강과 안전이 걱정됨
  - 항상 전화할 수는 없음
  - 과도한 알림은 피하고 싶음
- 니즈:
  - 오늘 상태 요약
  - 정말 중요한 상황만 알림
  - 빠르게 연락할 수 있는 수단

## 5. 문제 정의

### 현재 문제
1. 중장년층용 건강 앱은 정보는 많지만 실행이 어렵다.
2. AI 기능은 있어도 실제 생활 행동으로 연결되지 않는 경우가 많다.
3. 복약/운동/체크인 루틴이 오래 유지되지 않는다.
4. 보호자는 상시 확인이 어렵고, 본인은 감시받는 경험을 원치 않는다.
5. 응급이 아니더라도 불안/무기력/몸 상태 저하를 빠르게 도움받기 어렵다.

### 우리가 해결할 문제
- **복잡한 기능이 아니라 ‘오늘 바로 할 수 있는 행동’ 중심으로 설계한다.**
- **안전은 감시가 아니라 안심 경험으로 설계한다.**
- **AI는 진단이 아니라 생활 코칭과 동기 부여 역할을 맡는다.**

## 7. UX 원칙

### 중장년 친화 UX 원칙
- 큰 텍스트와 큰 터치 영역
- 한 화면 한 가지 핵심 행동
- 전문용어보다 생활 언어 사용
- 텍스트 + 아이콘 + 색상 중복 표현
- 되돌리기 쉬운 구조
- 과한 정보량 금지

### AI UX 원칙
- 긴 자유 대화보다 **선택형 코칭** 우선
- “왜 이걸 추천하는지” 짧게 설명
- 부담 없는 톤
- 의학적 진단처럼 보이지 않게 주의
- 실천 가능한 수준의 작은 행동 제안

### 안전 UX 원칙
- 공포가 아닌 안심 중심 카피
- 단계적 알림
- 예외 상황만 보호자에게 전달
- 사용자가 공유 범위를 통제 가능

## 8. 핵심 가치 제안

### 본 사용자에게
- 오늘 해야 할 일을 쉽게 알려준다
- 건강 관리가 어렵지 않게 느껴진다
- 위험하거나 불안할 때 바로 도움을 요청할 수 있다

- 부모님의 상태를 한눈에 이해할 수 있다
- 정말 필요한 상황만 알림받는다
- 연락과 대응이 빨라진다

## 10. 핵심 사용자 플로우

### Flow A. 아침 루틴 시작
1. 아침 알림 수신
2. 홈 진입
3. 오늘 할 일 확인
4. AI 코칭 확인
5. 첫 번째 행동 실행 (예: 물 마시기 / 스트레칭 / 복약)
6. 완료 체크

### Flow B. 컨디션 체크인
1. 체크인 요청 알림
2. 오늘 기분/몸 상태 선택
3. 필요 시 증상 선택
4. AI가 가벼운 코칭 제안
5. 기록 저장

### Flow C. 복약 누락 대응
1. 복약 알림 발송
2. 미확인 상태 지속
3. 재알림
4. 장시간 미확인 시 ‘확인 필요’ 상태 전환
5. 설정에 따라 보호자에게 알림

### Flow D. 위험/불안 상황 도움 요청
1. 사용자가 ‘도움이 필요해요’ 버튼 선택
2. 상황 선택
   - 몸이 불편해요
   - 넘어졌어요
   - 불안해요
   - 가족에게 연락하고 싶어요
3. 상황별 즉시 액션 제시
4. 보호자 연락 또는 긴급 연락

### Flow E. 보호자 확인
1. 보호자 앱 진입
2. 오늘 체크인/복약/상태 요약 확인
3. 이상 이벤트 여부 확인
4. 바로 연락

## 20. 최종 프로토타입 제작 지시문

다음 단계에서 제작할 프로토타입은 아래 조건을 만족해야 한다.

1. **모바일 중심 인터랙티브 앱 프로토타입**이어야 한다.
2. 본 사용자용 화면을 중심으로 하되, **보호자 핵심 화면 2~3개**를 포함한다.
3. 별도 **한국어 발표용 `/presentation` 페이지**를 포함한다.
4. UI는 중장년층 친화적으로 크고 명확해야 한다.
5. AI 코칭, 체크인, 복약 알림, 도움 요청, 보호자 연동이 핵심 플로우로 연결되어야 한다.
6. 실제 의료행위처럼 보이지 않도록 생활 코칭 서비스 톤을 유지해야 한다.
7. 결과물은 지원/제안용으로 바로 보여줄 수 있는 수준의 완성도를 목표로 한다.
